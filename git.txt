 SVN，git：代码管理（1.多人开发2.版本管理）
1.注意：先拉去，获取最新代码文件（把自己的代码备份，防止被覆盖）



新建项目时，用git 与项目关联
git init
git add README.md



git commit -m "first commit"
git remote add origin https://github.com/doubleyong/ReactDemo.git
git push -u origin master

有修改，用git上传(注：先拉取)
（git clone 项目的地址）克隆整个项目
git pull origin master 下载远程仓库的所有变动/////
git add 路径
git commit -m '注释'
git push -u origin master


一个新的项目 ：
成员先： git clone 分支地址   //拉取整个项目框架
每次提交前：git pull origin master //下载远程仓库的变动
然后推送给本地仓库：git add (./路径) // .是所有;路径是推送单个文件
加注释：git commit -m "注释内容" //推送给远程时必须先提交注释信息
推送给远程仓库：git push -u origin master //整个流程走完

注释：下一次推送给远程仓库时，除了第一步其他的步骤不变。

某文件取消修改删除后远程拉取恢复：
1、先将自己修改和添加的文件另存。
2、git fetch --all  //将远程仓库的最新内容全部拉到本地
3、git reset --hard origin/master //强制覆盖本地代码（与远程仓库代码保持一致）

注意：如果不使用第二步则会本地代码退回本地上一版本。
